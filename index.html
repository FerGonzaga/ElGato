Clases y Atributos

1. Clase IniciarJuego clase principal

Atributos:

  • J1: string - Representa al jugador 1, utilizando el símbolo "O".
  • J2: string - Representa al jugador 2, utilizando el símbolo "X".
  • tablero: List[string] - Lista de 9 elementos que representa el tablero, inicialmente vacío. Se utiliza para almacenar las jugadas de los dos jugadores.
    • [ 0 ] [ 1 ] [ 2 ]
    • [ 3 ] [ 4 ] [ 5 ]
    • [ 6 ] [ 7 ] [ 8 ]
  • turno_actual: string - Indica el turno actual, alternando entre "O" y "X". Comienza con el jugador 1 (J1).
  • root: Tk - La ventana principal del juego, que contiene el tablero y los controles.
  • ganador: Ganador - Una instancia de la clase Ganador, que se encarga de verificar si hay un ganador o empate.
  • mensaje: StringVar - Variable que almacena mensajes para mostrar información (como el estado del juego, ganador, o empate).

Métodos:

  • __init__(self, root: Tk) - Inicializa los atributos de la clase IniciarJuego y configura el entorno gráfico.
    • root es la ventana principal de Tkinter.
    • Se define quién empieza (turno_actual) y se llama a iniciar_juego() para configurar la interfaz del juego.
  • iniciar_juego(self) - Método que crea los botones del tablero de juego y organiza los elementos gráficos.
    • Crea un tablero de 3x3 con botones, donde cada botón corresponde a una casilla en el tablero.
    • Los botones están conectados al método ciclo() para ejecutar jugadas.
  • ciclo(self, posicion: int) - Controla el ciclo del juego:
    • Entrada: recibe la posicion del tablero (0-8) que corresponde a la casilla seleccionada.
    • Proceso:
      • Si la casilla está vacía, coloca el símbolo del jugador actual.
      • Guarda el movimiento en el tablero y verifica si hay un ganador o empate.
      • Si hay un ganador, se muestra el mensaje y se reinicia el juego después de 2 segundos.
      • Si no hay ganador, el turno se alterna entre los jugadores.
  • deshabilitar_botones(self) - Desactiva todos los botones cuando el juego ha terminado (ya sea por ganador o empate), para evitar nuevas jugadas hasta el reinicio.
  • reiniciar_juego(self) - Reinicia el estado del juego, restableciendo el tablero y habilitando los botones para una nueva partida.

2. Clase Ganador

Descripción:

Esta clase maneja la lógica para determinar si un jugador ha ganado o si el juego ha terminado en empate.

Métodos:

  • quien_gana(self, tablero: List[str]) -> bool:
    • Entrada: recibe el tablero como una lista de cadenas de 9 posiciones.
    • Proceso: compara las combinaciones ganadoras posibles (filas, columnas y diagonales) con el estado actual del tablero.
    • Salida: retorna True si alguna de las combinaciones ganadoras está completa con el mismo símbolo ("O" o "X"), o False si no hay ganador.
  • empate(self, tablero: List[str]) -> bool:
    • Entrada: recibe el tablero actual como una lista de cadenas.
    • Proceso: revisa si todas las posiciones del tablero están ocupadas.
    • Salida: retorna True si el tablero está lleno y no hay ganador, indicando un empate.

Combinaciones Ganadoras (8 Secuencias):

  • Filas:
    • [0, 1, 2]
    • [3, 4, 5]
    • [6, 7, 8]
  • Columnas:
    • [0, 3, 6]
    • [1, 4, 7]
    • [2, 5, 8]
  • Diagonales:
    • [0, 4, 8]
    • [2, 4, 6]

3. Clase Guardar

Método:

  • guardar(self, posicion: int, simbolo: string, tablero: List[str]) -> None:
    • Entrada: recibe la posicion en el tablero (0-8), el simbolo correspondiente al jugador actual ("O" o "X") y el tablero donde se guardará la jugada.
    • Proceso: coloca el símbolo en la posición especificada del tablero.
    • Salida: no tiene retorno, simplemente actualiza el estado del tablero.

Flujo del Juego

  1. Inicio del Juego:
    • Se crea una instancia de IniciarJuego.
    • Se inicializa el tablero con 9 posiciones vacías.
    • El primer turno se asigna a J1 ("O").
  2. Ciclo de Juego:
    • Entrada: El jugador actual selecciona una posición en el tablero.
    • Validación:
      • Se verifica si la posición seleccionada está vacía.
      • Si está vacía, se guarda el símbolo correspondiente ("O" o "X") utilizando el método guardar().
    • Comprobación de Ganador:
      • Se llama al método quien_gana() de la clase Ganador para verificar si el jugador actual ha ganado.
      • Si hay un ganador, se imprime un mensaje anunciando al ganador y el juego se reinicia tras 2 segundos.
    • Comprobación de Empate:
      • Si no hay ganador y todas las posiciones están ocupadas, se llama al método empate().
      • Si es un empate, se imprime un mensaje y el juego se reinicia tras 2 segundos.
    • Cambio de Turno:
      • Si no hay ganador ni empate, el turno se alterna al siguiente jugador y el ciclo se repite.
  3. Fin del Juego:
    • El juego finaliza si hay un ganador o si se reinicia tras un empate.
